# 三、分支操作规范

# 1. 分支命名规范

  • 功能分支：feature/功能描述
  • 发布分支：release/v版本号
  • 热修复分支：hotfix/问题简述

# 2. 远程分支操作

# 推送本地分支到远程（自动关联）
git push -u origin release/v12_0_0:personal/huwz/v12_0_0

# 删除远程分支
git push origin --delete person/huwz/v12_0_0_propery_view

# 强制回退到远程指定提交（谨慎使用！）
git reset --hard 2b7631cd81a915b380eeb4ff00dbbd1df492e176

# 四、高级协作技巧

# 1. 分支关联与同步

# 本地分支关联远程分支
git branch --set-upstream-to=origin/feature/web_tester feature/web_tester

# 变基更新（避免合并污染历史）

# 六、企业级最佳实践

  1. 分支保护策略

    • 主分支（main/develop）强制Code Review
    • 禁止直接git push -f（需管理员权限）

  2. 提交信息规范

    [类型] 简要描述（50字符内）
[空行]
详细说明（解决什么问题/影响范围）
关联任务：JIRA-123

    2

    5

    类型示例：feat, fix, docs, refactor

  3. 子模块更新原则

    • 子模块指向固定commit（避免自动跟踪分支）
    • 更新命令：git submodule update --remote --merge

# 七、灾难恢复技巧

场景 解决方案
误删未提交代码 git checkout -- <file>
提交到错误分支 git reset HEAD~ --soft → 切分支重提交
硬重置后找回代码 git reflog → 找回丢失的commit hash

⚠️ 高风险操作警示：

git reset --hard / git clean -df 不可逆！操作前务必：

  1. 确保工作区已提交或备份
  2. 使用 n 参数预览影响（如 git clean -ndf）

# 二、远程相关操作

1、本地代码提交到不同的远程分支提交处理方式

git push origin 本地分支名：远程分支名

如：git push origin release/v12_0_0:personal/huwz/v12_0_0

2、删除远程分支

git push origin --delete 远程分支名

如：git push origin --delete person/huwz/v12_0_0_propery_view

3、回退到最新代码分支处

Git reset --hard 远程分支SHA值

如·：git reset --hard 2b7631cd81a915b380eeb4ff00dbbd1df492e176 （commit SHA）

4.设置关联远端分支：

git branch --set-upstream-to=origin/remote_branch your_branch

5.git相关的常用命令

git cherry-pick 提交的hash值

例如：

a - b - c - d Master
e - f - g Feature 现在将提交f应用到master分支。

切换到 master 分支 $ git checkout master

Cherry pick 操作 $ git cherry-pick f

6.实现代码回退 git revert 远程分支SHA值
